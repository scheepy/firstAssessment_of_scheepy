概要
提供給使用者一個儲存Json字串的位置，根據不同的profile，同一個使用者可以存多筆資料。
單一筆資料最大為16MB。
 
連線
透過 Dispatcher & JobHandler 訂閱行情和取得回補資料。
連線(參閱AC文件)
*僅限SSL連線時可使用
開發機SSL連線:
hosts設定
172.28.16.205 test.icetech.com.tw
 
然後改用
wss://test.icetech.com.tw/ss2
test可換任意字元
 
dev環境可以直接連wss://ss2-dev.icetech.com.tw:10443/ss2
uat環境可以直接連wss://ss2-uat.icetech.com.cn:443/ss2
 
當連線成功後，將收到一份init字串。
登入AC
Dispatcher cmd範例:{"s":"ac","c":"login","r":"test123","d":{"COMPANY":"ICE" ,"PRODUCT":"SS2","LOGIN":"iceRoot","PWD":"xxxxxxxx","APP":"X3","RT_TRADINGS":true,"RT_PERMS":true,"RT_QRYTKS":1}}
[註]：COMPANY、PRODUCT、LOGIN、PWD四個欄位需自行帶入有權限的使用者資料
[註]：PWD必須為init與原始密碼進行動態編碼。(參閱AC文件)
 
登入成功後，可使用以下命令存取內容
接收格式固定為 {"s":"APConfig","c":"命令","d":條件,"r":"識別字串"}
 
使用命令帶有Company字節或Product字節，取得共用的設定(不是所有的設定)。
其中，不同company但具有相同product的使用者，使用getProduct等等命令時，將存取不同的設定。

命令權限
一般使用者只需要以下命令權限
sub、setUser、getUser、delUser、getProduct、getCompany
 
         貼牌客戶沒有BO可以操作調整設定
         因此client透過Ex命令組來達到BO調整設定的功能

命令：sub
條件：空Json字串
說明：使用者呼叫過此命令之後，會返回當前所有profile的版本號。並且在此之後，單一profile更新版本，都會收到通知。使用者須自行決定要不要主動呼叫更新。
例：{"s":"APConfig","c":"sub","d":{},"r":"test"}

回傳格式
 Key  說明
 s  APConfig
 c  sub
 d  紀錄所有profile的版本
 Key  說明
 c  (Commpany)Json字串。Key為profile，value為版號。
 p  (Product)
 u  (User)
 cd  錯誤代碼
 ErrorCode  說明
 0  成功
 r  使用者帶入的r
例：{''s":"APConfig", "c":"sub", "d":{ "c":{"default":1}, "p":{"default2":2, "default3":4}, "u":{"default5":6} }, "cd":0, "r":"rid001"}
命令 : subEx
說明：訂閱指定商品，呼叫此命令之後，會返回該商品所有profile的版本號，並在之後，單一profile更新版本，將會收到通知。
條件：Json字串
 Key  類型  說明
 product  String  欲訂閱的商品。
例：{"s":"APConfig","c":"subEx ","d":{"product":"aaa"}},"r":"anyString"}

回傳格式
 Key  說明
 s  APConfig
 c  search
 d  資料Josn
Key  說明
 cpuf  Json字串，使用者代上來的company(c)、product(p)、user(u)、profile(f)
 data  Json字串組成的array，內容為company、product、user、profile
 msg  當cd不為0時的錯誤說明
 cd  錯誤代碼
 ErrorCode  說明
 0  成功
 r  使用者帶入的r
例：{"s":"APConfig","c":"search","d":{"cpuf":{"company":"as"},"data":[{"company":"as","profile":"default"},{"company":"as","product":"yc-master","profile":"default"},{"company":"as","product":"yc-master","user":"user001","profile":"default"},{...}]},"r":"123"}

命令： search
說明：用來查詢符合條件的使用者名稱與相關的所有設定
條件：Json字串
 Key  類型  說明
 product  String  任意字串。預設為空值，可代入"*"來代表任意值。
 user  String  任意字串。預設為空值，可代入"*"來代表任意值。
 profile  String  任意字串。預設為空值，可代入"*"來代表任意值。
例：{"s":"APConfig","c":"search ","d":{"profile":"anyString","data":{ (略) }},"r":"123"}

回傳格式
 Key  說明
 s  APConfig
 c  search
 d  資料Josn
Key  說明
 cpuf  Json字串，使用者代上來的company(c)、product(p)、user(u)、profile(f)
 data  Json字串組成的array，內容為company、product、user、profile
 msg  當cd不為0時的錯誤說明
 cd  錯誤代碼
 ErrorCode  說明
 0  成功
 r  使用者帶入的r
例：{"s":"APConfig","c":"search","d":{"cpuf":{"company":"as"},"data":[{"company":"as","profile":"default"},{"company":"as","product":"yc-master","profile":"default"},{"company":"as","product":"yc-master","user":"user001","profile":"default"},{...}]},"r":"123"}

命令： setCompany、setProduct、setUser、setExCompany、setExProduct、set ExUser
條件：Json字串
 Key  類型  說明
 profile  String  任意字串。預設為default。
 data  Json  要儲存的資料
 product  String  只在Ex的情況下作用。直接指定對象。
 user  String  只在Ex的情況下作用。直接指定對象。
例：{"s":"APConfig","c":"setUser","d":{"profile":"anyString","data":{ (略) }},"r":"threadID"}

回傳格式
 Key  說明
 s  APConfig
 c  setCompany 或 setProduct 或 setUser
 d  資料Josn
Key  說明
 msg  訊息
 cd  錯誤代碼
 ErrorCode  說明
 0  成功
 r  使用者帶入的r

 cd  錯誤代碼

 0  成功
 r  使用者帶入的r
